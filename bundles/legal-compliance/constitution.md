# Constitution - Legal & Compliance

## Principios Generales

1. **Compliance First** - Toda feature debe cumplir con las regulaciones aplicables antes de implementación.
2. **Privacy by Design** - La privacidad debe considerarse desde el inicio de cada spec.
3. **Auditability** - Todas las decisiones críticas deben tener trazabilidad.
4. **Risk-Based Approach** - Mayor escrutinio en features que manejen datos sensibles, pagos o IA.

## Guardrails Obligatorios

- No recolectar datos personales sin consentimiento explícito y documentado.
- Toda integración con terceros debe tener DPA (Data Processing Agreement) revisado.
- Usar licencias open source compatibles con el modelo de negocio.
- Mantener registro de versiones de políticas legales (ToS, Privacy Policy).
- Features con IA deben declarar si usan datos de entrenamiento y si hay bias risks.
